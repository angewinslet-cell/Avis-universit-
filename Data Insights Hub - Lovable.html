⚠️ Important : Lovable est conçu pour des apps React/TypeScript déployées sur l'infrastructure Lovable Cloud (Cloudflare Workers). Je ne peux pas créer une app Python ni la déployer sur un serveur Python externe (Streamlit Cloud, PythonAnywhere, Railway, Fly.io, etc.) depuis cet environnement.

Ce que je peux faire pour votre TP

Option A — Vous écrire le code Python (Streamlit ou Flask) prêt à déployer Je vous génère un fichier .py complet + requirements.txt + instructions de déploiement (Streamlit Community Cloud, Hugging Face Spaces, Fly.io, Railway…). Vous le déployez en quelques clics et obtenez votre lien public. Je peux livrer ces fichiers dans /mnt/documents/ à télécharger.

Option B — Construire l'app équivalente ici (React + Lovable Cloud) Même fonctionnalités (formulaire, base de données, dashboard, export CSV), déployée automatiquement sur *.lovable.app en cliquant sur Publier. Robuste, fiable, mais ce n'est pas du Python.

👉 Pour un TP exigeant du Python, je recommande l'Option A avec Streamlit + Streamlit Community Cloud (gratuit, lien public, hors Render, très simple).

comment va se passer le déploiement étapes par étapes

Déploiement sur Railway — Guide étape par étape

📋 Prérequis

  • Un compte GitHub (gratuit) → https://github.com
  • Un compte Railway (gratuit) → https://railway.app (connexion via GitHub)
  • L'archivestreamlit_app.zipque je t'ai généré

🚀 Étape 1 : Préparer le code sur GitHub

  1. Télécharge streamlit_app.zipdepuis le panneau de droite et dézippe-le sur ton ordinateur.
  2. Allez sur github.com → cliquez sur "New deposit" (bouton vert).
  3. Donne un nom :avis-universite(par exemple) → laisse en Public → cliquez sur "Créer un dépôt" .
  4. Sur la page du repo, cliquez sur "uploader un fichier existant" .
  5. Glissez-déposez TOUS les fichiers du dossier dézippé (app.py,requirements.txt,Procfile,runtime.txt,README.md,.gitignore).
  6. Cliquez sur « Valider les modifications » .

🚂 Étape 2 : Créer le projet sur Railway

  1. Va sur Railway.app → cliquez sur "Connexion" → connectez-vous avec GitHub .
  2. Cliquez sur "Nouveau projet" → choisis "Déployer depuis le dépôt GitHub" .
  3. Autoriser Railway à ton GitHub → sélectionne le repoavis-universite.
  4. Railway détecte automatiquement Python et lance le build (attend ~2 min).

⚙️ Étape 3 : Configurer les variables d'environnement

Dans ton projet Railway, cliquez sur le service → onglet "Variables" → ajoute :

VariableValeurRôle
MOT DE PASSE ADMINTonMotDePasseFort123!Mot de passe du tableau de bord admin
DATA_DIR/donnéesDossier de la base SQLite
MAX_TENTATIVES_DE_CONNEXION5Nombre d'essais avant verrouillage
VERROUILLAGE_SECONDES300Durée du verrouillage (5 min)

💾 Étape 4 : Ajouter un volume persistant (CRITIQUE)

⚠️ Sans cette étape, les données SQLite seront perdues à chaque calcul.

  1. Dans ton service → onglet « Paramètres » → rubrique « Volumes » .
  2. Cliquez sur « Nouveau volume » .
  3. Chemin de montage :/données(doit correspondre exactement àDATA_DIR).
  4. Confirmer → Le chemin de fer redémarre automatiquement.

🌐 Étape 5 : Générer le lien public

  1. Onglet "Paramètres" → section "Réseau" .
  2. Cliquez sur « Générer le domaine » .
  3. Railway crée une URL du type :https://avis-universite-production.up.railway.app
  4. C'est ton lien d'application en ligne ✅

✅ Étape 6 : Tester l'application

  1. Ouvre l'URL → tu vois le formulaire d'avis.
  2. Remplis et soumets un avis test.
  3. Cliquez sur l'onglet "Dashboard Admin" → entre tonMOT DE PASSE ADMIN.
  4. Vérifiez que le graphique de régression linéaire s'affiche.
  5. Teste l' export CSV/Excel .

🔄 Mises à jour futures

Pour modifier l'application : éditez les fichiers sur GitHub → Railway redéploie automatiquement en ~1 min.

💡 Astuces

  • Plan gratuit Railway : 5 $ de crédit/mois (~500h d'exécution) — suffisant pour un TP.
  • Logs : onglet "Déploiements" → cliquez sur le déployé actif pour voir les logs en temps réel.
  • Erreur 502 : attend 30s après le déploiement, Streamlit met du temps à démarrer.

Guide de déploiement Railway en 6 étapes fourni : GitHub → Railway → variables → volume persistant → domaine public → test.
